How It Works

Zero-knowledge control.

Cr sits between your AI agent and the blockchain, enforcing policy without ever touching your private keys.

1
Agent requests an action
Your AI agent submits a signed intent — e.g. "transfer 0.5 ETH to address X" — through the Cr SDK.
2
Policy engine evaluates rules
Every rule you've defined — spend caps, allowlists, time windows — is checked in under 50 ms against the intent payload.

Key Architecture

We never hold
your keys. Period.

Cr uses Multi-Party Computation (MPC) and ERC-4337 smart accounts to give AI agents the power to transact without centralising key material anywhere.

  • ◆
    Threshold Signature Scheme (TSS)

    Key material is split into shards held by you, your agents, and optional recovery parties — requiring a t-of-n quorum to sign.

  • ◆
    Smart Account Policies (ERC-4337)

    On-chain session keys with granular permissions for each agent — automatically expiring, revocable, and scoped to specific contracts or amounts.

  • ◆
    Zero-Knowledge Audit Proofs

    Every enforcement decision produces a ZK proof — verifiable on-chain without revealing sensitive policy details.

// cr.config.ts — define agent policies import { definePolicy } from '@cr/sdk' export default definePolicy({ agent: 'trading-bot-v2', wallet: '0xYourWallet…', // spending limits limits: { daily: { usd: 5_000 }, single: { usd: 500 }, }, // only these contracts allowlist: [ '0xUniswapRouter', '0xAavePool', ], // time-based gating schedule: { tz: 'UTC', hours: [8, 22], // 08:00–22:00 }, // approval chain for large txns approvals: { threshold: { usd: 1_000 }, signers: ['0xOwner', '0xTreasury'], minSigs: 1, }, })
